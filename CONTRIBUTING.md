# Contributing Guidelines

Thank you for your interest in contributing to VibePilot Web!

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/vibepilot-web.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Run tests and linting: `npm run lint`
6. Build to ensure no errors: `npm run build`
7. Commit with clear messages: `git commit -m 'Add feature description'`
8. Push to your fork: `git push origin feature/your-feature`
9. Open a Pull Request

## Code Style

- Use TypeScript for type safety
- Follow ESLint rules (auto-fixed with `npm run lint`)
- Write descriptive variable and function names
- Add comments for complex logic
- Keep components small and focused

## Pull Request Process

1. Update README.md if adding features
2. Ensure all tests pass
3. Provide a clear description of changes
4. Link any related issues
5. Request reviews from maintainers

## Security

- Never commit `.env.local` or secrets
- Report security issues privately
- Follow security best practices in code
